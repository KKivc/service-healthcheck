## Phase 1: MVP 基础版 (已完成)

**目标：** 跑通核心链路，满足个人使用

### 功能
- ✅ HTTP 服务健康检查
- ✅ SQLite 存储 + 历史记录查询
- ✅ Web 状态看板（表格 + 状态卡片）
- ✅ 命令行定时检查模式
- ✅ 飞书 Webhook 告警
- ✅ Docker 容器化部署

### 关键决策
- 选择 SQLite 而非 MySQL → 零运维，个人场景足够
- 状态变更才告警 → 避免告警风暴
- CLI + Web 双模式 → 灵活部署

---

## Phase 2: 智能增强 (当前阶段)

**目标：** 让监控从"显示状态"进化到"诊断问题"

### 功能
- [x] ✅ AI 故障根因分析
- [x] ✅ 延迟趋势折线图
- [x] ✅ 健康评分体系
- [x] ✅ 深色模式
- [x] ✅ UI 添加/删除服务
- [x] ✅ 前端筛选（正常/异常/全部）
- [ ] 📋 自定义检查间隔（按服务）
- [ ] 📋 通知渠道扩展（邮件、企业微信）

### 关键决策
- LLM API 通过环境变量配置 → 兼容 OpenAI、DeepSeek 等多种模型
- AI 诊断同步生成 → 检查完立即展示，不需要二次请求

**目标：** 从个人工具到可分享的状态服务

---

## 迭代原则

1. **用户反馈驱动** — 功能优先级来自实际使用痛点
2. **做减法 > 做加法** — 不堆功能，每个新增必须有明确场景
3. **可逆优先** — 先做容易改的决策，验证后再投入
